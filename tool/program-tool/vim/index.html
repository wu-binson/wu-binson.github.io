vim

vim无疑是编辑器的王者，可定制性极其强大，结合各类插件可以打造成功能齐全的IDE，缺点是学习难度较大，对新手不友好。

vim的入门可以通过在终端键入vimtutor来学习基本操作，这是官方出品，当然是最好的入门教程，可能没有之一。进阶的教程可以看看vimjc 这个网站。

vim如果要自己配置.vimrc，将要花费很多时间精力，象oh my zsh一样，vim-spf13为我们提供一个配置的方法，简单无脑配置后， 便拥有一个强大版本的vim了。

重要插件

下面将列举一些重要的插件，有些被包含在vim-spf13，有些没有。每种插件的使用教程可以在vim里键入;help [插件名称]之后看到。

vundle

Vundle是一个Vim插件管理器，用于方便地安装、更新和卸载插件。

  1. 安装新插件:找到插件的github网址，放进.vimrcm里面，再打开vim，键入:PluginInstall的命令便会自动安装。
  2. 卸载插件:先在.vimrc删掉相关插件的配置信息，再打开vim，键入:PluginClean命令。
  3. 更新插件:在vim里键入:PluginUpdate

nerdtree

nerdtree是vim的分屏和文件导航插件，效果图如下： 如图所示，左栏显示当前目录，用户可以在上面切换目录，查看文件存不存在，或者打开文件。右边两栏则是左右分屏的文件编辑栏。

easymotion

easymotion则是在vim的基本跳转功能的基础上增加了新的跳转命令，用户在普通模式下键入w(其中默认是逗号)或者b, 便可看到类似下面的画面: 再依次键入红色或者黄色处的键值，便能跳到该处的词首。

vim-signature

除了easymotion这种跳转，用户还可以安装vim-signature插件。在普通模式下，用户可以键入比如”ma”的命令，这时vim的左边便会出现’a’的红色符号: 表示在这行有了一个mark，用字母a标记。然后在其他地方便可以依据这个mark跳转到这个标记位置，当然用户还可以增加其他字母标记的mark或者删除这些mark。

ctags

ctags的跳转则是针对编程而言，在函数或者变量处按下’Ctrl+]’便可以跳到定义处，再按下’Ctrl+t’便可以跳回原处。

使用ctags插件的前提是机器也要安装ctags，然后在相关目录下输入’ctags -R’, 让当前目录下所有ctags支持的语言格式文件生成tags。

tabnine

tabnine是一个强大且支持大多数流行语言的补全插件，与传统的补全工具不同，tabnine会根据用户的习惯进行补全，而且可以长句地补全，效率非常高。

vim vs visual studio code

visual studio code是最近崛起且还在快速发展的IDE，拥有丰富的插件系统，安装配置简单，使用方便，学习难度小。短期接触后也比较喜欢，但个人还是觉得 vim的效率更高，且占用内存小，启动更快，还有一个其他IDE无法比拟的优势，就是远程到服务器，仍然可以在终端命令行使用vim。
